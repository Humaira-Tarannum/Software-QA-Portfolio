# Software Requirements Specification (SRS)

## Project Name
ShopEasy - E-commerce Web Application

## Objective
The objective of ShopEasy is to provide customers with a secure and user-friendly platform to browse, purchase and manage products online.

## Modules

1. User Registration
2. User Login
3. Product Search
4. Product Details
5. Shopping Cart
6. Wishlist
7. Checkout
8. Payment
9. Order History
10. User Profile

## Functional Requirements

### Registration
- User should be able to register with a valid email.
- Password must contain at least 8 characters.

### Login
- Registered users should be able to log in.
- Invalid credentials should display an error message.

### Product Search
- Users can search products by name.
- Users can filter products by category and price.

### Cart
- Users can add, update and remove products.

### Payment
- Users should be able to complete payment using supported methods.

### Order History
- Users should be able to view previous orders.
